  • Fale 3 formas de diminuir o page load (tempo de carregamento real e percebido).
    1. Limpeza do código: Remover elementos e espaços na página desnecessários, utilizar-se de imagens mais leves para que sejam carregadas de imediato e boa organização e estruturação do código são práticas que contribuem para um carregamento mais limpo e tranquilo.
    2. Reutilização de Conteúdo: Garantir que conteúdos com importância relevante sejam guardados e reutilizados via memória cache.
    3. Ferramentas de compressão: Existem várias ferramentas de compressão de código que possibilitam um carregamento mais rápido do mesmo pelo browser. Muito utilizado para arquivos JavaScript.

  • Considere o HTML5 como uma plataforma web aberta. Quais são os blocos de construção de HTML5?

    • <header>: Define o cabeçalho da página;
    <nav>: Define o menu de navegação;
    <footer>: Define o rodapé do elemento;

    <section>: Define uma seção do layout em um determinado elemento. Pode conter um header e footer;

    <article>: Consiste em uma composição independente de conteúdo com um cabeçalho e algum texto que faça sentido até de forma isolada;

  • Liste quantas propriedades display você puder lembrar.
    1. Display: none; Elementos com Display:none não são renderizados.
    2. Display: block; Elementos carregados como bloco, sendo posicionados um abaixo do outro. Não permite elementos HTML ao lado.
      (Ps: exceto com a propriedade float).
    3. Display: inline; Elementos dentro do bloco na mesma linha, sendo posicionado lado a lado. Seu tamanho varia com o tamanho do conteúdo.
    4. Display: inline-block; Elemento é posicionado inline, ou seja, na mesma linha do conteúdo adjacente, mas detém comportamento de bloco.
    5. Display: table; Elemento formatado como uma tabela, contendo as mesmas manipulações de table-row e table-data.
    6. Display: list-item; Elemento será formatado como o item de uma lista, aparecendo até mesmo o marcador (bullet), na frente de cada um dos itens.

  • Qual a diferença entre inline e inline-block?

    • Inline: Um elemento do tipo Inline não aceita nenhum tipo de espaçamento (padding/margin) ou formatações de tamanho (height, width). Estes ocupam somente o espaço necessário para sua exibição.

    Inline-block: Já um elemento Inline-block é a junção entre inline e block e pode sofrer tais alterações redimensionais, aceitando atributos css como height, width, padding, margin, para sua configuração.


  • Qual a diferença entre elementos posicionados de forma relativa, fixa, absoluta e estática?

    • Relativa: Você consegue alinhar o elemento a ele mesmo. Você move o elemento baseado na sua posição inicial, podendo utilizar as propriedades top, right, bottom e left para executar a ação.

    Absoluta: Quando um elemento recebe a posição absoluta, ele automaticamente deixa de preencher o fluxo padrão do elemento. Se o mesmo estiver dentro de uma div, por exemplo, ele com a posição absoluta se comportaria como se não estivesse dentro de uma. Pode-se utilizar também as propriedades top, right, bottom e left para movimentá-la.(Ps: o elemento fica absoluto em relação ao body).

    Fixa: Se comporta de forma semelhante à Absoluta, com a diferença de que se comporta referente ao viewport do browser. Ao movimentar a scroll-bar da página, por exemplo, notamos que o elemento permanece na posição que lhe foi designada, mesmo que a página esteje em movimento. Muito usado em menus de sites inclusive.

    Estática: O posicionamento estático se comporta perante ao valor padrão de posicionamento. Os elementos permanecem onde estão e uso dessa declaração serve para sobrescrever um posicionamento declarando anteriormente no código.


  • Qual a diferença entre .call e .apply?

    • Os dois são métodos JavaScript que solucionam o mesmo problema: generalização de uma ação por meio de uma função para evitar a repetição de código.

    .call: No método call, utiliza-se do parâmetro this.(contexto) e os seguintes são parâmetros da função. Ex: função.call(parametro,arg);
    .apply: O método apply é quase a mesma coisa. A diferença é que se utiliza de Arrays nos parâmetros. Ex: função.apply(parametro,[argArray]);


  • Qual a diferença entre == e ===?

    • Igual(==): Compara o conteúdo entre variáveis para verificar se eles são iguais.
    Ex: String '2' == int 2 (Verdadeiro).

    Idêntico(===): Compara o conteúdo entre variáveis para verificar se eles são iguais juntamente com seus tipos.
    Ex: String '2' === int 2 (Falso) | int 2 === int 2 (Verdadeiro).
